protocolo de comunicacion en la web: HTTP, el backend es un directorio aparte, el frontend es el directorio que representa la aplicacion web, la interfaz de usuario.

El backend es el servidor que proporciona los datos y funcionalidades a la aplicacion web.
El frontend es el cliente que consume los datos y funcionalidades del backend.

El backend se encarga de la logica de negocio, la base de datos, la autenticacion, la autorizacion, la seguridad, la escalabilidad, la performance, la compatibilidad con diferentes tipos de dispositivos y navegadores.

js, go, python, java, ruby, php, c#(se pueden usar para backend )

el back y el fron deben entender HTTP, entra en juego un concepto llamado API, (aplication programing interface)
 
 es una serie de reglas y protocolos para permitir la comunicacion entre dos o mas sistemas

2. 
CRUD (Create, Read, Update, Delete).

create: crear recurso
read: leer recurso
update: actualizar recurso
delete: eliminar recurso

equivalencias http: create=POST, read=GET, update=PUT, delete=DELETE

codigos de respuesta http: 200, 201, 204, 400, 404, 500

200: ok, todo bien
201: creado, recurso creado
202: accepted, recurso aceptado
204: no contenido, no hay contenido
400: bad request, errores en el cliente
401: unauthorized, no autorizado
403: forbidden, prohibido
404: not found, no encontrado
500: internal server error, error en el servidor
501: not implemented, no implementado
502: bad gateway, error en el gateway
503: service unavailable, servicio no disponible
504: gateway timeout, timeout en el gateway
505: http version not supported, version de http no soportada
