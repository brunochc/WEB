Entrevistador 2 FASE 1: 1.¿Qué son las Single Page Applications (SPA) y cómo funcionan en React? R: Las Single Page Applications (SPA) son aplicaciones web que cargan una sola página y actualizan su contenido dinámicamente sin recargar la página completa. En React, las SPA se implementan utilizando componentes y el Virtual DOM para actualizar solo las partes necesarias de la interfaz de usuario. 2.¿Cuál es la diferencia entre el state y las props en React? R: El state es un objeto que contiene datos que pueden cambiar durante la ejecución del programa y que se actualizan mediante eventos o acciones del usuario. Las props son datos que se pasan de un componente padre a un componente hijo y que no pueden ser modificados por el hijo. 3.¿Qué hace la propiedad position: absolute y cómo se comporta respecto al padre? R: La propiedad position: absolute establece que un elemento se posicione en una ubicación específica dentro de su contenedor padre, independientemente de la posición de los otros elementos. 4.¿Qué hace useEffect y cuándo se ejecuta? R: useEffect es un hook que permite ejecutar código después de que un componente se renderiza en la pantalla. Se ejecuta después del renderizado del componente y puede ser utilizado para realizar operaciones como la obtención de datos de una API o la actualización del estado del componente. 5.¿Qué unidades existen en CSS y cuál es la diferencia entre px, em y %? R: Las unidades existentes en CSS son px (píxeles), em y % (porcentaje). La diferencia entre px y em es que px es una unidad fija que mide el tamaño de un pixel en la pantalla, mientras que em es una unidad relativa que mide el tamaño de la fuente del elemento padre. La diferencia entre px y % es que px mide el tamaño en píxeles, mientras que % mide el tamaño en porcentaje del tamaño del elemento padre.
